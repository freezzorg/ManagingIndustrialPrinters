Измени код существующего у меня android-приложения для ТСД (Теминал Сбора Данных) со сканером штрих-кодов (qr-кодов), которое управляет базой данных промышленных принтеров так, что бы он соответствовал следующей логике работы:
Приложение реализовано на Flutter/Dart.

Примерные характеристики ТСД:
  Android 10;
  RAM 3 GB;
  ROM 32 GB;
  2D Imager;
  4.0";
  480 x 800.

Сервис PrintComm, работающий с базой данных ObjectBox, написан на Go и имеет HTTP-сервис для обмена данными. Файл main.go находится в каталоге temp в каталоге сервиса.

Приложение должно иметь функции:
  Приложение не должно хранить ни какие данные;
  Привязка принтера к линии. Статус принтера "В работе";
  Привязка принтера к линии. Статус принтера "Подключен";
  Отвязка принтера от линии. Статус принтера "Не в работе";
  Возможность ввода данных вручную;
  Возможность добавлять и удалять принтеры в базу;
  Возможность редактировать данные принтеров (только поля: Номер принтера; Код модели принтера; IP-адрес принтера; Порт принтера);
  Возможность показ списка принтеров с их данными.

Основная работа приложения сводится к сканированию последовательно штрих-кодов (qr-код) (предложи как лучше организовать современный и удобный визуалный интерфейс исходя из разрешения экрана ТСД).

Привязка принтера к линии. Статус принтера "В работе": Сканируется штрих-код (qr-код) линии, сканируется штрих-код (qr-код) принтера со статусом "В работе".
  При этом,
  Если в базе ObjectBox сервиса PrintComm принтер до этого имел статус "Не в работе":
    принтеру присваивается статус "В работе" (поле Status);
	присваивается уникальный идентификатор линии к которой принтре привязывается (поле UID);
	присваивается наименование линии (поле RM).
  Если в базе ObjectBox сервиса PrintComm принтер до этого имел статус "В работе" или "Подключен" (случай, когда забыли отвязать принтер от линии при замене) выдаётся предупреждение, что принтер привязан к такой-то линии, с двумя кнопками: "Отмена" и "Перепривязать" (предложи альтернативу).
    В случае нажатия кнопки "Перепривязать":
      принтеру присваивается статус "В работе" (поле Status);
  	  присваивается уникальный идентификатор линии к которой принтре привязывается (поле UID);
  	  присваивается наименование линии (поле RM). В случае нажатия кнопки "Отмета" очищаем заполняемые поля формы.
	
Привязка принтера к линии. Статус принтера "Подключен": Сканируется штрих-код линии (qr-код), сканируется штрих-код (qr-код) принтера со статусом "Подключен". Дальнейшая работа аналогична привязке со статусом "В работе".

Отвязка принтера от линии. Статус принтера "Не в работе": Сканируется штрих-код (qr-код) принтера со статусом "Не в работе".
  При этом,
  Если в базе ObjectBox сервиса PrintComm принтер до этого имел статус "Не в работе" просто очищаем поля и переходим в исходное состояние;
  Если в базе принтер до этого имел статус "В работе" или "Подключен":
    принтеру присваивается статус "Не в работе" (поле Status);
	присваивается уникальный идентификатор линии 00000000-0000-0000-0000-000000000000 (поле UID);
	очищаем наименование линии (поле RM).

Номер принтера, код модели принтера, IP-адрес принтера, порт принтера не изменяются.

Структура, описывающая данные, содержащиеся в базе ObjectBox:
type Printer struct {
  Id     uint64 json:"id,omitempty" objectbox:"id"                		// Уникальный идентификатор (автоинкремент)
  Number int    json:"number" objectbox:"index" validate:"gte=1,lte=99" // Номер принтера
  Model  int    json:"model" validate:"gte=0,lte=9"                    	// Код модели принтера
  IP     string json:"ip" validate:"required,ipv4"                		// IP-адрес принтера
  Port   string json:"port" validate:"required,max=5"             		// Порт принтера
  UID    string json:"uid" objectbox:"index" validate:"len=36"    		// Уникальный идентификатор линии (UUID)
  RM     string json:"rm,omitempty" validate:"omitempty,max=9"    		// Код RM (опционально)
  Status int    json:"status" validate:"gte=0,lte=9"              		// Код статуса принтера
}
Поле UID содержит Уникальный идентификатор производственной линии по маркировке медного и оптического кабеля из  из 1С.
Поле RM содержит номер производственной линии по маркировке медного и оптического кабеля из 1С.
Поля Number, Model, IP, Port, Status задаются в android-приложении.

Каждая линия имеет свой штрих-код (qr-код), содержащий: уникальный идентификатор линии, наименование линии, вид выполняемой операции.
Например: 59809bc5-4330-11eb-93e7-005056a0a885, PM01, Линия наложения оболочки; 5263e93d-5c10-11ed-999a-005056a0d8c7, PM69, Линия наложения оболочки микрокабеля.
